**OrganizationNoteBack** — серверная часть (бэкенд) для системы ведения рабочих заметок и встреч. Обеспечивает централизованное хранение данных, синхронизацию между устройствами и бизнес-логику для клиентского мобильного приложения [WorkNoteBook](https://github.com/Ensin1031/WorkNoteBook).

---

## Функциональные возможности

Бэкенд реализует REST API для следующих операций:  

1. Управление пользователями
   - Регистрация и аутентификация
   - Управление профилем (администрирование пользователей)

2. Работа с заметками (Notes)
   - Создание, чтение, обновление, архивирование, мягкое удаление заметок (с возможностью восстановления администратором системы).
   - Поддержка структуры связи заметок пользователей к встречам

3. Работа со встречами (Meetings)
   - Создание и управление записями о встречах, в том числе архивирование и мягкое удаление (с возможностью восстановления администратором системы).
   - Привязка заметок к конкретным встречам.

4. Синхронизация данных
   - Предоставление клиентам дельты изменений на основе временных меток (`updated_at`).
   - Разрешение конфликтов по времени последнего изменения.
   - Поддержка гибридной модели хранения (клиенты могут хранить данные локально и синхронизировать с сервером).

## Инструкция по развёртыванию локально

### Предварительные требования
- **Python 3.12+** [скачать](https://www.python.org/downloads/)
- **Git** [скачать](https://github.com/) либо [скачать](https://git-scm.com/)
- **SQLite** (опционально, для разработки)

### Шаги по запуску

1. **Клонируйте репозиторий**
    ```commandline
    git clone https://github.com/Ensin1031/OrganizationNoteBack.git
    cd OrganizationNoteBack
    ```
   
2. ***Создайте и активируйте виртуальное окружение***
    ```commandline
    sudo apt install python3.12 python3.12-venv
    python3.12 -m venv .venv
    . .venv/bin/activate
    ```

3. ***Установите зависимости***
    ```commandline
    pip install -r requirements.txt
    ```

4. ***Настройте переменные окружения***    
    В базовой версии, если устраивают стандартные настройки - не требуется.
    Создайте файл **.env** в корне проекта
    ```bash
    APP_TITLE="API системы ведения рабочих заметок и встреч"
    APP_DESCRIPTION="Серверная часть (бэкенд) для системы ведения рабочих заметок и встреч"
    APP_VERSION="0.0.1"
    DEBUG=true
    SECRET_KEY="12345"
    DOCS_URL="/swagger"
    REDOC_URL="/redoc"
    SQLITE_URL="./work_notebook.db"
    ROOT_PATH="/api/v1"
    APP_TIME_ZONE=""
    ```
   
5. ***Запуск сервера (локальный запуск)***
    ```commandline
    uvicorn app.main:app --host 0.0.0.0 --reload --port 5000
    ```
    Для остановки всех процессов на порту работы микросервиса
    ```commandline
    sudo fuser -k 5000/tcp
    ```
   
6. Работа с API:
   - SWAGGER:
      ```commandline
      http://127.0.0.1:5000/api/v1/swagger
      ```
   - REDOC:
      ```commandline
      http://127.0.0.1:5000/api/v1/redoc
      ```
